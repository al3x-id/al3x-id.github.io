Nigeria Fintech Analysis - PowerBI Project

Project Overview

An interactive Power BI dashboard analyzing ₦71M in Nigerian fintech transactions. The project uncovers customer trends by age, channel, and transaction type; highlights peak periods and failure rates; and provides actionable business insights to improve reliability and customer engagement.

Objectives

  1. What’s the trend of transaction volume and value across time (monthly)?
  2. What age group transacts the most and through what channels?
  3. Are there any patterns in failed transactions (e.g., by platform, location, transaction type)?
  4. Is there a peak period (month/day) for financial transactions?
  5. Which transaction type is used the most?
  6. What is the average transaction amount by transaction type?
  7. Which platform/channel processes the most transactions in volume and value?
  8. What percentage of transactions were successful, failed, or pending?
  9. What does this data reveal about customer behavior and business performance? What would you advise the management to do?

Dataset

Size: Contains 7000 rows of Nigeria Fintech Data from the year 2023- 2024

Key Columns: TransactionID, CustomerID, CustomerAge, Gender, Location, TransactionType, Chanenl, Amount, TransactionDate, Status.

ETL Process (Extract, Transform and Load)

This process involved obtaining the data, loading to power query for cleaning and transformed it for analysis. The cleaning involves the formatting of key columns such as dates and currency.

Calculated Metrics Using DAX

The following Calculated Metrics were derived using DAX formula to drive decision-making.

Calculated Columns

  1. Month
    • DAX Formula:
      • 
Month = FORMAT('Nigerian Fintech'[TransactionDate].[Month],"YYYY-mm")
    • This is use to extract month name from the TransactionDate Column.
  2. MonthNo
    • DAX Formula:
      • 
MonthNo = MONTH('Nigerian Fintech'[TransactionDate].[Date])
    • This is use to extract month NO. from the TransactionDate column (month name is sort by this column to keep them in order during analysis)
  3. Age Group
    • DAX Formula:
      • 
Age Group = SWITCH(
    TRUE(),
    [CustomerAge] <= 25, "18-25",
    [CustomerAge] <= 35, "26-35",
    [CustomerAge] <= 45, "36-45",
    [CustomerAge] <= 59, "46-59",
    "60+"
)
    • This is use to categorize the CustomerAge into different age bracket.
  4. Day
    • DAX Formula:
      • 
Day = FORMAT('Nigerian Fintech'[TransactionDate].[Date],"dddd")
    • This is use to extract day of the week name from the TransactionDate column.
  5. DayNo
    • DAX Formula:
      • 
DayNo = WEEKDAY('Nigerian Fintech'[TransactionDate].[Date],2)
    • This is use to extract day NO. from the TransactionDate column (day name is sort by this column to keep them in order during analysis)

Calculated Measures

  1. Average Amount
    • DAX Formula:
      • 
AverageAmount = AVERAGE('Nigerian Fintech'[Amount])
  2. Failure Rate(%)
    • DAX Formula:
      • 
FailureRate (%) = 
DIVIDE(
    CALCULATE(COUNTROWS('Nigerian Fintech'),'Nigerian Fintech'[Status] = "Failed"),
    COUNTROWS('Nigerian Fintech')
) * 100
  3. Formatted Total Amount
    • DAX Formula:
      • 
Formatted Total Amount = 
VAR Num = SUM('Nigerian Fintech'[Amount])
RETURN
SWITCH(
    TRUE(),
    Num >= 1000000000, FORMAT(Num / 1000000000, "0.00") & "B",
    Num >= 1000000, FORMAT(Num / 1000000, "0.00") & "M",
    Num >= 1000, FORMAT(Num / 1000, "0.00") & "K",
    FORMAT(Num, "0"))
  4. Transaction Value
    • DAX Formula:
      • 
Transaction Value = SUM('Nigerian Fintech'[Amount])
  5. Transaction Volume
    • DAX Formula:
      • 
Transaction Volume = COUNT('Nigerian Fintech'[TransactionID])

Modelling, Analysis and Visualization

KPI

  • Total Transaction: 7000 – This is the total transactions made over the two years.
  • Total Transaction Value: ₦71.01M – The total transaction value over the two years.
  • Average Transaction Value: ₦10.14k – The average transaction value over the two years.
  • Failure Rate:10.30% – Percentage of failed Transactions.

Trend of transaction volume and value across time (monthly)

  • Both transaction volume and value fluctuate monthly.
  • Peaks in value and volume occur in May and July each above ₦6.4M in value.
  • Lows appear around February, June, and November when value dips below ₦5.6M.

Insight: Transaction activity is strongest mid-year, while early and late months see reduced activity.

Which age group transacts the most and through what channels?

  • Age group 46–59 has the highest transaction volume.
  • Followed by age groups 36–45 and 26–35.
  • The 46–59 group leans heavily on Agent and Mobile App, whereas younger groups are more evenly spread across channels.

Insight: Older users (46–59) are the most active, especially on Agents

Patterns in failed transactions (platform, location, type)

  • Highest failure rates: Mobile App (11.7%), Agent (10.4%), and POS (10.3%).
  • Lower rates: Web (9.6%) and USSD (9.6%).

Insight: Mobile App and Agent platforms face higher failure rates, indicating potential infrastructure or user experience challenges.

Peak period (day) for financial transactions:

  • Highest volumes: Monday (1031) and Friday (1025).
  • Lowest: Thursday (957).

Insight:Start and end of the workweek have higher activity.

Which transaction type is used the most?

  • Highest volume: Payment (1213 transactions).
  • Close behind: Transfer (1193) and Deposit (1189).

Insight: Payments are the most frequent transaction type.

Average transaction amount by transaction type

  • Highest: Transfer (₦10.5K).
  • Lowest: Loan Repayment and Withdrawal (₦9.9K).

Insight: Transfers are typically higher in value than other types.

Which platform/channel processes the most transactions in volume and value?

  • Highest volume: Web (1480) and Agent (1445).
  • Highest value: Agent (₦15M) and Web (₦15M).

Insight: Web and Agent handle the most transactions and value.

Percentage of transactions: successful, failed, pending

  • Successful: 84.79%
  • Failed: 10.30%
  • Pending: 4.91%

Insight: Most transactions succeed; however, over 16% still encounter issues.

Dashboard

What the data reveals

Customer behavior:

  • Channel use is balanced: Customers actively use multiple channels (Web, Agent, POS, USSD, Mobile App) almost evenly for transactions.
  • Transaction type spread: Payments, transfers, and deposits lead transaction volume, but the average transaction amount is slightly higher for transfers and airtime.
  • Age group: The 46–59 age group has the highest transaction volume, showing older users remain very engaged. Even younger groups (18–25, 26–35) transact steadily, though less than older users.
  • Day of week: Activity is quite balanced across weekdays, peaking on Monday and Friday, and slightly dropping on Thursday and weekends.

Business performance:

  • Healthy total volume & value: ₦71.01M total value and 7000 transactions is strong.
  • Average transaction value: ₦10.14K is modest, suggesting a mix of high and low-value transactions.
  • Failure rate: Overall failure rate is 10.3%, with the highest on Mobile App (11.7%). USSD and Web perform better (~9.6%).
  • Seasonal trend: Transactions peaked in January and July (~₦6.5M value, >600 transactions) and dipped in February, June, and November (₦5.2M–₦5.4M).

Insights & Recommendations

  • Reduce failure rates – especially on Mobile App and Agent channels:
    • Review technical performance, server reliability, and user interface flows.
    • Run usability tests to identify common user errors.
    • Strengthen backend infrastructure or partner service integrations that fail.
  • Targeted campaigns:
    • Age-focused: Engage younger age groups (18–25) more with loyalty rewards, social media promotions, and student-friendly features.
    • Transaction type: Boost lower-volume services (e.g., loan repayment, airtime) with bundle offers or discounts.
  • Channel optimization: Encourage use of more stable channels (USSD, Web) for critical transactions and invest in improving the Mobile App for easy access.
  • Even out monthly dips: Offer seasonal promotions or campaigns in February, June, and November to smooth out transaction dips, and align product launches to these slower months.
  • Improve average transaction value: Cross-sell products, upsell premium services, and encourage higher basket sizes through cashback or discount offers.
